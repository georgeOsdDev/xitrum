WebSocketActor

trait WebSocketActor extends Actor with Logger

An actor will be created when there's request. It will be stopped when: - The connection is closed - WebSocket close frame is received or sent

Abstract Value Members

  1. abstract def execute(action: Action): Unit

    action

    The action just before switching to this WebSocket actor. You can extract session data, request headers etc. from it, but do not use respondText, respondView etc. Use these to send WebSocket frames: - respondWebSocketText - respondWebSocketBinary - respondWebSocketPing - respondWebSocketClose

    There's no respondWebSocketPong, because pong is automatically sent by Xitrum for you.
